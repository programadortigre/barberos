Guía Básica para Usar Git y GitLab

Esta guía es para personas que no saben nada de Git ni GitLab.
Sigue paso a paso para entender desde lo más básico.

1. ¿Qué es Git?

Git es una herramienta que ayuda a guardar y controlar los cambios que haces en tus archivos de código o documentos. Es como un "historial" donde puedes ver qué cambiaste, cuándo y quién.

Para usar Git, necesitas instalarlo en tu computadora.

¿Cómo instalar Git?
- En Windows: descarga desde git-scm.com
- En macOS: usa Homebrew con brew install git
- En Linux (Ubuntu): sudo apt-get install git

2. Configura Git en tu PC

Después de instalar, abre la terminal (o consola) y escribe estos comandos para configurar tu nombre y correo:

git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@example.com"

Esto es para que Git sepa quién está haciendo los cambios.

3. ¿Qué es GitLab?

GitLab es una página web donde puedes guardar tus proyectos con Git. Además, permite que tú y otras personas trabajen juntos en el mismo proyecto.

Es como una carpeta en internet donde tienes la última versión de tu código y pueden ver los cambios.

Ir a GitLab

4. Crear una cuenta en GitLab

  1. Entra a https://gitlab.com/users/sign_up
  2. Regístrate con tu correo o usando Google/GitHub
  3. Verifica tu correo y accede

5. Crear un nuevo proyecto en GitLab

  1. En tu cuenta, haz clic en New project
  2. Selecciona Blank project
  3. Escribe un nombre para tu proyecto, ejemplo: mi-primer-proyecto
  4. Escoge si quieres que sea público o privado (para empezar, privado está bien)
  5. Haz clic en Create project

6. Clonar el proyecto a tu computadora

Para trabajar con Git, necesitas bajar ("clonar") el proyecto a tu PC. Para eso:

  1. En la página del proyecto, haz clic en Clone y copia la URL que aparece (usa HTTPS para empezar).
  2. Abre la terminal (o consola) en tu PC.
  3. Escribe este comando (cambia la URL por la que copiaste):
git clone https://gitlab.com/tu-usuario/mi-primer-proyecto.git

Esto creará una carpeta en tu PC con el proyecto.

7. Hacer cambios y subirlos a GitLab

Ahora haz algún cambio (por ejemplo, crea un archivo README.md).

Para subir los cambios, usa estos comandos:

git add .
git commit -m "Mensaje que describe tus cambios"
git push origin main

Si la rama principal no es main, puede ser master. Puedes usar git push origin master en ese caso.

8. Trabajar con ramas (opcional, recomendado para proyectos en equipo)

Las ramas te permiten hacer cambios sin afectar la versión principal hasta que estés listo.

Ejemplo para crear y usar una rama nueva:

git checkout -b mi-rama-nueva
git add .
git commit -m "Cambios en mi rama"
git push origin mi-rama-nueva

Luego puedes ir a GitLab y hacer un Merge Request para unir los cambios a la rama principal.

9. Merge Request en GitLab

Cuando subes una rama, GitLab te mostrará un botón para crear un Merge Request.

  1. Haz clic en Create merge request
  2. Escribe un título y descripción clara para explicar tus cambios
  3. Haz clic en Submit
  4. Cuando el equipo revise y apruebe, se pueden fusionar los cambios a la rama principal

10. Consejos finales

  • Siempre escribe mensajes claros en los commits.
  • Haz commits pequeños y frecuentes.
  • Practica usando Git y GitLab en proyectos pequeños.
  • No tengas miedo de preguntar o buscar tutoriales.

¡Con práctica te volverás un experto!
